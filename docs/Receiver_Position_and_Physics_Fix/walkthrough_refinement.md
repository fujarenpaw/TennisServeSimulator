# 修正内容の確認 - 物理演算の精度向上とUI改善

高い打点での軌道計算を改善し、ターゲットマーカーの操作性を向上させました。

## 実施した追加の修正

### 1. 物理演算の最適化（低弾道優先）
- **改善点**: これまでの「最小誤差」優先から、「許容誤差（15cm以内）を満たす中で最も低い（打ち下ろす）角度」を優先するように `PhysicsEngine.ts` を変更しました。
- **効果**: 打点高さを3mなどに設定した際、以前のように山なりのロブ軌道になるのではなく、直線的に着弾点を狙うプロのようなサーブ軌道が計算されるようになりました。

### 2. UIの改善（ターゲットマーカーの拡大）
- **変更内容**: `src/graphics/TrajectoryVisualizer.ts` にて、着弾点を示す緑のゴーストマーカーの半径を `0.2` から `0.4` へ拡大しました。
- **効果**: マウスや指でのドラッグ操作が非常にしやすくなっています。

## 検証結果

### 物理演算の挙動
- **3m打点での確認**: 打点3m、サービスライン付近への着弾を設定した場合に、直線的な鋭い軌道が生成されることを確認しました。
- **150km/h制限との整合性**: 速度上限を守りつつ、可能な限り低い軌道が選択されています。

### UIの操作性
- ターゲットマーカーが大きくなり、視認性とドラッグの反応が向上したことを確認しました。
