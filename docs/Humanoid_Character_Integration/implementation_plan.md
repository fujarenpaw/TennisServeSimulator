# 高度なプロシージャルアニメーションによる現実味の向上計画

現在の簡易的な可動モデルを改良し、全身の連動（キネティック・チェーン）を意識したリッチなモーションを実装します。

## 提案される変更

### 1. 関節構造の拡張 (`HumanoidCharacter` の更新)
現在は肩と肘のみですが、以下の関節を追加して「身体のしなり」を表現します。
- **Spine (脊椎)**: 前後の反り、左右の捻りを追加。
- **Hips (股関節/骨盤)**: 体重移動を表現。
- **Knees (膝)**: サーブのタメ（屈伸）やレシーブの低い姿勢を実現。
- **Wrists (手首)**: ラケットのしなりを表現。

### 2. モーションの刷新

#### サーブ (Serve)
- **トスアップ**: 身体を弓なりに反らせ、トロフィーポーズ（タメの姿勢）を強調。
- **インパクト**: 膝の伸展と連動させ、高い打点へ向かう動きを動的に。
- **フォロースルー**: ラケットが身体の反対側まで振り抜かれる大きな動き。

#### レシーブ & 移動 (Receive & Movement)
- **サイドシャッフル**: 前を向いたまま横に移動するテニス特有の足運び。
- **スプリットステップ**: サーブが打たれる直前の軽いジャンプ（準備動作）。
- **体幹の捻り**: スイングの始動時に肩を入れ、腰を捻る動作を追加。

### 3. 同期の深化
ボールの速度や到達時間に合わせて、スイングの「タメ」の時間を動的に調整し、常にジャストミートしているように見せます。

## 検証計画
- [ ] サーブ時に膝の屈伸と背中の反りが見えるか。
- [ ] 横移動時に「カニ歩き」ではなくテニスらしいシャッフルになっているか。
- [ ] スイング後、慣性で身体が少し前や横に流れるような自然なフォロースルーになっているか。

> [!TIP]
> 外部の重いアニメーションデータを読み込む代わりに、数式（サイン波やイージング）を組み合わせた制御を行うことで、軽量さを維持しつつ「生きて動いている」ような質感を追求します。
