# 修正内容の確認

## 概要
以下の修正を行いました。
1. **ボール軌道の改善**: 軌道が垂直に落下する問題を修正。
2. **ネット衝突判定**: ボールがネットを通過する際、高さが足りない場合はネットで止まるように変更。
3. **コート寸法表示**: チェックボックスON時に、コート上の主要な寸法を表示するように変更。

## 検証手順

### 1. 軌道生成の確認
- アプリケーションを起動してください。
- サーブパラメータをデフォルトのままにし、「計算実行」などを押してサーブを打ちます。
- **期待される動作**:
  - ボールの軌道が、リリース地点からバウンド地点まで滑らかな放物線（に近い曲線）を描くこと。
  - バウンド直前に、これまでのような不自然な「真下への落下」がないこと。

### 2. ネット衝突の確認
- `trajectoryHeight` を 2.0m 以下、`bounceDepth` を短めに設定するなどして、ネットに掛かるようなサーブ設定にします（例: height=0.5m）。
- **期待される動作**:
  - 軌道がネットの位置（z=0）で完全に停止し、相手コート側に線が伸びないこと。

### 3. 寸法表示の確認
- UI上の「コート寸法を表示（Show Dimensions）」チェックボックスをONにしてください。
- **期待される動作**:
  - コートの白線から少し離れた位置に、水色（シアン）の寸法線と数値が表示されること。
  - 数値がコート中央ではなく、外側に配置され見やすくなっていること。
  - **サイドラインの数値（Y方向）が、サーバー側から見やすい向き（以前と逆の180度回転）になっていること**。
  - 以下の主要寸法が表示されていること：
    - ベースライン幅: 10.97m (手前)
    - サービスライン幅: 8.23m (奥)
    - サイドライン（左）: **11.89m** (ネット〜ベースライン)
    - サイドライン（右）: 6.40m (ネット〜サービス), 5.49m (サービス〜ベース)
    - **センターラインの数値は表示されていないこと**。

### 4. ボールの視認性
- アニメーションを再生します。
- **期待される動作**:
  - ボールが以前より大きく（半径0.2）、黄色ではっきり見えること。

## 変更ファイル
- `src/core/PhysicsEngine.ts`: 軌道計算ロジック、衝突判定
- `src/graphics/CourtVisualizer.ts`: 寸法ラベル表示機能
